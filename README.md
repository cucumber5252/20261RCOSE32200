# 20261RCOSE32200 Shell Project

간단한 Unix-like shell을 C 언어로 구현한 프로젝트입니다.
사용자로부터 명령어를 입력받아 파싱한 뒤, 외부 명령어 실행, built-in 명령어 처리, 입출력 리다이렉션, 백그라운드 실행, 단일 파이프 기능을 지원합니다.

## 주요 기능

### 1. 사용자 입력 처리

- 표준 입력으로부터 명령어를 입력받습니다.
- 입력 문자열 끝의 newline 문자를 제거합니다.
- 빈 입력은 무시하고 다음 입력을 기다립니다.

### 2. 명령어 파싱

- `strtok`을 사용하여 공백과 탭 기준으로 명령어를 분리합니다.
- 명령어와 인자를 `argv` 배열 형태로 저장합니다.

### 3. 외부 명령어 실행

- `fork()`를 사용하여 자식 프로세스를 생성합니다.
- 자식 프로세스에서 `execvp()`를 사용하여 외부 명령어를 실행합니다.
- foreground 명령어의 경우 부모 프로세스가 `waitpid()`로 자식 프로세스가 종료될 때까지 기다립니다.
- 명령어가 비정상 종료된 경우 non-zero exit status를 출력합니다.

### 4. Built-in 명령어

다음 built-in 명령어를 지원합니다.

- `exit`
    - shell을 종료합니다.

- `cd`
    - 현재 작업 디렉토리를 변경합니다.
    - 인자가 없는 경우 `HOME` 디렉토리로 이동합니다.

- `pwd`
    - 현재 작업 디렉토리 경로를 출력합니다.

### 5. 백그라운드 실행

명령어 뒤에 `&`를 붙이면 백그라운드에서 실행됩니다.

예시:

```bash
sleep 5 &
```

백그라운드 실행 시 부모 프로세스는 자식 프로세스를 기다리지 않고, 생성된 프로세스의 PID를 출력합니다.

### 6. 입출력 리다이렉션

기본적인 입력 및 출력 리다이렉션을 지원합니다.

입력 리다이렉션 예시:

```bash
cat < input.txt
```

출력 리다이렉션 예시:

```bash
ls > output.txt
```

리다이렉션은 `open()`, `dup2()`, `close()`를 사용하여 처리합니다.

### 7. 단일 파이프

하나의 파이프 연산자 `|`를 지원합니다.

예시:

```bash
cat input.txt | grep hello
```

파이프 실행 시 두 개의 자식 프로세스를 생성하고, 왼쪽 명령어의 표준 출력을 오른쪽 명령어의 표준 입력으로 연결합니다.

## 빌드 방법

다음 명령어로 컴파일할 수 있습니다.

```bash
gcc -Wall -Wextra -o shell shell.c
```

## 실행 방법

컴파일 후 다음 명령어로 shell을 실행합니다.

```bash
./shell
```

실행하면 다음과 같은 프롬프트가 출력됩니다.

```bash
$
```

프롬프트에 원하는 명령어를 입력하면 됩니다.

## 실행 예시

```bash
pwd
cd ..
ls -al
cat < input.txt
ls > output.txt
cat input.txt | grep hello
sleep 5 &
exit
```

## 구현 범위

이 shell은 다음 기능을 중심으로 구현되었습니다.

- 사용자 입력 처리
- 명령어 파싱
- 외부 명령어 실행
- built-in 명령어 처리
- foreground/background 실행
- 기본적인 입출력 리다이렉션
- 단일 파이프 처리

복잡한 quoting 처리, 여러 개의 파이프 연결, built-in 명령어에 대한 리다이렉션, 복합 명령어 처리 등은 포함하지 않습니다.
